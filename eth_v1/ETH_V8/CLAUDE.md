## ETH/USDT V8.16 자동매매 봇

### 전략 개요
- **심볼**: ETHUSDT
- **타임프레임**: 10분봉 (5분봉 수집 → 10분봉 리샘플링)
- **Fast MA**: EMA(250)
- **Slow MA**: EMA(1575)
- **마진**: 계정 잔액의 20% (복리)
- **레버리지**: 10x
- **필터**: ALL OFF (ADX/RSI/EMA Gap 없음)

### SL/TA/TSL 설정
| 항목 | 값 | 비고 |
|------|-----|------|
| SL | 2.0% | 진입가 대비 역행 시 손절 |
| TA | 54% | 가격 기준 +54% 도달 시 TSL 활성화 |
| TSL | 8.0% | 고점 대비 -8% 하락 시 익절 |

### ★ 가격 기준 (반드시 준수)
| 항목 | 기준 | 설명 |
|------|------|------|
| **진입** | **봉마감 (종가)** | EMA cross 확인 후 **1봉 뒤 진입** (i > ws) |
| **SL 발동** | **실시간 가격** (30초) + 저가/고가 | 빠른 손실 관리 |
| **SL 청산가** | **SL가 (slp)** | 종가가 아닌 설정된 SL 가격 |
| **TA 활성화** | **고가/저가** (intrabar) | 봉 중간에 +54% 도달해도 활성화 |
| **TSL 고점추적** | **실시간 가격** + 고가/저가 | TSL 활성 후에만 |
| **TSL 청산** | **실시간 가격** (30초) + 종가 | 빠른 수익 보호 |
| **REV 판단** | **봉마감 (종가)** | EMA cross 확인 |

### ★ 진입 타이밍 (핵심)
```
크로스 발생 봉: watching 시작, ws = i
진입 조건: i > ws (크로스 발생 봉에서는 진입 안 함)
→ 최소 1봉 (10분) 후 진입

검증 결과: 1봉 지연이 즉시 진입보다 우수
  1봉 지연: PF 1.75, MDD 37.8%
  즉시 진입: PF 1.67, MDD 40.0%
```

### ★ SL/TSL 우선순위
```
TSL 미활성 (tsl_active=False):
  → SL만 작동 (저가/고가 + 실시간 가격)
  → SL 발동 시 즉시 청산 (TSL/REV 체크 안 함)

TSL 활성 (tsl_active=True):
  → SL 비활성 (SL 체크 건너뜀)
  → TSL만 작동 (종가 + 실시간 가격)
  → TSL 미발동 시 REV 체크
```

### GPU/CUDA 백테스트 가속 규칙

#### 환경
- **GPU**: NVIDIA GeForce RTX 4060 Ti 8GB
- **CUDA Toolkit**: 12.6 (`C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v12.6`)
- **Numba**: 0.64.0 (JIT + CUDA 지원)
- **CuPy**: 14.0.1
- **Python**: 3.12

#### 백테스트 최적화 시 필수 규칙
1. **대량 파라미터 최적화** (1,000개 이상 조합)는 반드시 **Numba CUDA GPU**로 실행할 것
2. **단건 백테스트**는 **Numba @njit** (CPU JIT)로 실행할 것 (0.001초/건)
3. **실시간 트레이딩 봇**은 기존 Python 그대로
4. GPU 사용 시 **배치 크기 1,500** (8GB VRAM 기준), 초과 시 자동 분할

#### 새 백테스트/최적화 스크립트 작성 시
1. `backtest()` 함수는 **Numba @njit** 데코레이터 적용
2. dict.get() 대신 **개별 float 인자**로 전달
3. 문자열/Boolean → **숫자 코드**로 변환 (예: dual_mode='BOTH' → 1.0)
4. 대량 최적화는 `numba.cuda` 커널로 작성, `run_cuda_batch()` 패턴 사용
5. 결과 검증: 기존 Python 버전과 **1:1 비교** 후 사용
