```

```

### 4.3 RSI 설정

| 파라미터 | 범위 |
|----------|------|
| RSI 기간 | 7, 14 |
| Long 진입 | RSI 하한 30~40, 상한 60~75 |
| Short 진입 | RSI 하한 30~45, 상한 65~75 |

### 4.4 볼린저 밴드 (엔진 C 활용)

```
BB Width = (Upper - Lower) / Middle × 100
→ 변동성 레짐 분류 (수축기/확장기)

BB %B = (Close - Lower) / (Upper - Lower)
→ Long: BB_%B < 0.8 (밴드 상단 초과 매수 방지)
→ Short: BB_%B > 0.2 (밴드 하단 초과 매도 방지)
```

### 4.5 거래량 프로파일 (엔진 C)

```
Volume MA Ratio = Volume / SMA(Volume, 20)
→ 1.5 이상: 강한 거래량 (진입 신뢰도 ↑)
→ 0.5 이하: 약한 거래량 (진입 억제)
```

### 4.6 ATR (변동성 측정)

```
ATR Period: 7, 14
ATR Multiplier (SL용): 2.5~4.0 (엔진 C)
ATR Multiplier (TP용): 3.0~6.0
ATR Multiplier (Trail용): 2.5~4.0
```

## 6. 엔진 C 청산 및 리스크 관리

### 6.1 ATR 기반 동적 손절

| 파라미터 | 엔진 C |
|----------|--------|
| ATR 배수 | 2.5~4.0 |
| 예상 SL 범위 | 3~8% |

```
Long SL = Entry_Price - ATR(14, 5m) × ATR_SL_Multiplier
Short SL = Entry_Price + ATR(14, 5m) × ATR_SL_Multiplier

최소 SL: 1% | 최대 SL: 8% (레버리지 10x 기준)
```

### 6.2 분할 청산 구조

```
TP1: ATR × 3~4 도달 → 30% 물량 청산 → SL을 손익분기점으로
TP2: ATR × 5~6 도달 → 30% 물량 청산 → SL을 TP1 수준으로
TP3: 트레일링 스톱으로 나머지 40% 관리
```

### 6.3 트레일링 스톱

```
Trail_Distance = ATR(14, 5m) × Trail_ATR_Multiplier (2.5~4.0)
Long: Trail_SL = Highest_High - Trail_Distance
Short: Trail_SL = Lowest_Low + Trail_Distance

단계별 강화:
  ROI 5%+ → Trail 5%
  ROI 10%+ → Trail 4%
  ROI 15%+ → Trail 3%
  ROI 20%+ → Trail 2.5%
```

### 6.4 시간 기반 청산 (엔진 C 전용)

```
최대 보유 시간: 24~72시간 (최적화 대상)
보유 시간 초과시:
  ROI > 0: 전량 청산 (수익 확보)
  ROI < 0: SL 수준으로 청산 (손실 제한)
```

### 6.5 역신호 청산

```
현재 LONG 보유 중 + SHORT 진입 조건 발생:
  → LONG 전량 청산 (시장가) → SHORT 즉시 진입

기존 포지션 ROI > 10%인 경우: 50% 부분 청산 후 나머지 트레일링
역신호 강도가 낮은 경우 (Score < 40): 무시
```
