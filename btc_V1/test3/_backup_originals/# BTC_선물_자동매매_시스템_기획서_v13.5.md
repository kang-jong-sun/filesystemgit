## 4. SL -7%가 최적인 이유

### 4-1. 강제청산 거리와 SL의 관계

```
10x 격리마진 강제청산 거리: 10%

SL -7%: 여유 3%p → 75개월 중 1회 강제청산 → 복리 보전
SL -8%: 여유 2%p → 3회 강제청산 → 복리 일부 손실
SL -9%: 여유 1%p → 7회 강제청산 → 복리 심각 손실
```

### 4-2. 강제청산의 실제 비용 (격리마진)

```
격리마진 강제청산 = 해당 포지션의 증거금만 손실

$10,000 잔액 기준:
증거금: $10,000 × 20% = $2,000
강제청산 시: $2,000 손실 (잔액의 20%)

vs SL -7%:
포지션: $2,000 × 10x = $20,000
SL 손실: $20,000 × 7% = $1,400 (잔액의 14%)

차이: 강제청산(-20%) vs SL(-14%) = 6%p
```

SL -7%은 SL 발동 횟수가 SL -9%보다 많지만, 강제청산을 거의 피하므로 **복리 기반이 보전**됩니다.

## 10. 지표 구현

### ADX (Wilder's Smoothing 필수)

```python
adx = dx.ewm(alpha=1/14, min_periods=14, adjust=False).mean()
# rolling().mean() 사용 금지!
```

### RSI (Wilder's Smoothing)

```python
gain_avg = gain.ewm(alpha=1/14, min_periods=14, adjust=False).mean()
loss_avg = loss.ewm(alpha=1/14, min_periods=14, adjust=False).mean()
rsi = 100 - 100 / (1 + gain_avg / loss_avg)
```

### 진입 코드

```python
cross_up = (ema7 > ema100) and (prev_ema7 <= prev_ema100)
cross_dn = (ema7 < ema100) and (prev_ema7 >= prev_ema100)

if cross_up and adx >= 30 and 30 <= rsi <= 58:
    if protection.can_enter():
        margin = protection.get_margin(balance)
        size = balance * margin * 10
        enter_long(size)
        set_stop_loss(entry_price * 0.93)  # SL -7%
```

### 청산 코드

```python
# 1. SL -7% (바이낸스 서버)
# 2. 트레일링
if peak_pnl >= 0.08:
    trailing_active = True
if trailing_active:
    trail_sl = peak_price * 0.94  # 롱 기준 고점 -6%
    if close <= trail_sl:
        close_position()

# 3. 역신호 (필터 필수)
if holding_long and cross_dn and adx >= 30 and 30 <= rsi <= 58:
    close_long()
    enter_short()  # 보호 조건 통과 시
```
