## 멀티TF 순차진입 구조

```
교차마진 단일 계좌 $5,000

포지션 없을 때 순차적으로 신호 확인:
  1순위: 30m WMA(3)/EMA(200) ADX(20)>=35 RSI 35~60 크로스
  2순위: 15m EMA(7)/EMA(250) ADX(14)>=45 RSI 35~75 크로스
  3순위: [TF3] 60m EMA(3)/EMA(100) ADX(20)>=30 RSI 30~70 크로스  ← 이 문서

→ 어떤 TF에서든 신호 발생 시 단일 포지션 진입
→ TSL 청산 후 3캔들 뒤 추세 지속 시 리엔트리

결과: 18거래(단일TF) → 115거래(멀티TF+리엔트리) = 6.4배 증가
```

**TF3 (60m)은 3순위 신호원으로, TF2와 TF1에서 신호가 없을 때 발동합니다.**
**가장 낮은 ADX 임계값(30)으로 거래빈도 확보를 담당합니다.**

## TF3 (60m) 전략 설정

### 핵심 파라미터

```python
# ═══ TF3 60m 신호 (3순위) ═══
TF3_TIMEFRAME = '60m'
TF3_MA_FAST   = ('EMA', 3)
TF3_MA_SLOW   = ('EMA', 100)
TF3_ADX       = (20, 30)    # ADX period=20, threshold=30
TF3_RSI       = (30, 70)    # RSI min=30, max=70
```

### TF3 특성 비교

| 항목 | TF1 (15m) | TF2 (30m) | TF3 (60m) |
|------|-----------|-----------|-----------|
| MA Fast | EMA(7) | WMA(3) | **EMA(3)** |
| MA Slow | EMA(250) | EMA(200) | **EMA(100)** |

**TF3 (60m)의 ADX 30은 세 TF 중 가장 낮은 임계값 — 거래빈도 최대화 역할.**

## TF3 특성: Slow MA EMA(100)

TF3은 EMA(100)을 사용합니다 (TF1: EMA(250), TF2: EMA(200)과 비교):

| TF | Slow MA | 기간(60m 환산) | 특성 |
|----|---------|---------------|------|
| TF1 (15m) | EMA(250) | ~250 × 15m = 62.5시간 | 장기 필터 |
| TF2 (30m) | EMA(200) | ~200 × 30m = 100시간 | 중장기 필터 |
| TF3 (60m) | EMA(100) | ~100 × 60m = 100시간 | 중기 필터 |

EMA(100) @ 60m는 약 4.2일 평균으로, 중기 추세를 포착합니다.

## TF3 (60m) 진입 조건

### 진입 로직

```
[TF3 60m 신호 발생 조건 — 3순위]

1. TF2 (30m) 신호 없음 (1순위 우선)
2. TF1 (15m) 신호 없음 (2순위 우선)
3. 60분봉 기준:
   - EMA(3) > EMA(100) 골든크로스 → LONG
   - EMA(3) < EMA(100) 데드크로스 → SHORT
4. ADX(20) >= 30 (가장 낮은 임계값)
5. RSI 30~70 범위 내 (가장 넓은 범위)
6. 동일방향 스킵 (SKIP_SAME_DIR=True)
7. 월간 손실 한도 미도달
```

### TF3 ADX 30의 역할

ADX(20) >= 30은 세 TF 중 가장 완화된 조건:
- 중간 강도의 추세에서도 진입 허용
- 거래빈도 확보 (TF2/TF1가 놓친 기회 포착)
- 단, 60분봉이라는 더 긴 TF가 노이즈를 자연 필터링

### 청산 우선순위 (공통)


```
TSL 청산 후 3캔들(180분) 뒤:
  - 60m 추세 동일 방향 지속 중
  - ADX(20) >= 30 재충족
  - RSI 30~70 재충족
  → 재진입 허용
```
