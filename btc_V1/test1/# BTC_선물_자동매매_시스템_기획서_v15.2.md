### 5-2. PF >= 8의 구조적 한계

```
PF = 총이익 / 총손실

PF >= 8을 달성하려면:
  - 방법 1: 손실을 극단적으로 줄임 → SL을 넓게(10%+) + 강제청산 허용 (v14.3 방식)
    문제: FL 27회 = 강제청산 시 증거금 100% 손실이 PF에서 "SL 손실"로 계산 안 됨
    실제: 강제청산 27회 × 30% 증거금 = 복리 기반 810% 손실
    결론: PF가 높아 보이지만 실제 리스크는 극히 높음

  - 방법 2: 거래 수를 극단적으로 줄임 → 월 0.3회 미만
    문제: 통계적 유의성 부족 (75개월에 20회 미만)
    결론: 과최적화 위험 극대

  - 방법 3: 보호 메커니즘으로 손실 거래 차단
    문제: 월간한도/3연패정지로 손실 거래를 건너뛰면 PF 상승
    실제: 건너뛴 거래가 수익이었을 경우 기회비용 발생
    결론: PF 수치는 올라가지만 실제 수익률 하락 가능
```

### 5-3. 정직한 PF 기준

```
FL 0회 + 통계적 유의 거래수(50회+) 조건에서:

  30분봉 PF 상한: ~5.8 (P1에서 확인)
  보호 적용 PF 상한: ~6.5
  지연진입 적용 PF 상한: ~4.1

  → FL 0 + 거래 50회+ 조건에서 PF >= 8은 현 데이터 기간에서 비현실적
  → PF >= 8을 요구하면 FL 허용 또는 극소 거래수 필요
```

## 10. HMA(7)/EMA(200) 계산 공식

### EMA(3) - 초단기 이동평균

```python
# Exponential Moving Average (span=3)
EMA(3) = close.ewm(span=3, adjust=False).mean()

# EMA(3)의 특성:
# - 가격 변화에 매우 빠르게 반응
# - EMA(7)보다 크로스 빈번 → 6캔들 지연으로 필터링
# - ADX>=35와 조합하여 강한 추세만 포착
```

### EMA(200) - 장기 추세선

```python
EMA(200) = close.ewm(span=200, adjust=False).mean()
```

### ADX(14) - Wilder's Smoothing

```python
def adx(high, low, close, period=14):
    plus_dm = high.diff().where((high.diff() > -low.diff()) & (high.diff() > 0), 0)
    minus_dm = (-low.diff()).where((-low.diff() > high.diff()) & (-low.diff() > 0), 0)
    tr = max(high-low, abs(high-close.shift()), abs(low-close.shift()))
    atr = tr.ewm(alpha=1/period, min_periods=period, adjust=False).mean()
    plus_di = 100 * plus_dm.ewm(alpha=1/period).mean() / atr
    minus_di = 100 * minus_dm.ewm(alpha=1/period).mean() / atr
    dx = 100 * abs(plus_di - minus_di) / (plus_di + minus_di)
    return dx.ewm(alpha=1/period, min_periods=period, adjust=False).mean()
```

### RSI(14) - Wilder's Smoothing

```python
def rsi(series, period=14):
    delta = series.diff()
    gain = delta.where(delta > 0, 0)
    loss = (-delta).where(delta < 0, 0)
    avg_gain = gain.ewm(alpha=1/period, min_periods=period, adjust=False).mean()
    avg_loss = loss.ewm(alpha=1/period, min_periods=period, adjust=False).mean()
    rs = avg_gain / avg_loss
    return 100 - (100 / (1 + rs))
```

### v14.3이 PF >= 8인 이유

```
SL = -10% (발동 0회)
강제청산 거리 = 10% (10x 레버리지)

→ SL이 강제청산과 같은 수준이므로 SL 대신 강제청산 발생
→ 강제청산 손실은 "증거금 전액 손실" (잔액의 30%)
→ PF 계산에서 SL 손실이 0이므로 총손실이 매우 작게 산출
→ PF = 총이익 / (역신호 손실만) = 매우 높은 수치

실제 리스크:
  강제청산 27회 x 증거금 30% = 810%의 복리 기반 손실
  MDD 72% = 계좌가 72% 감소하는 구간 존재
  2020년 -62.5%, 2021년 -8.2% (초기 2년 손실)
```
