## 3. 지표 상세 (Indicator Formulas)

### 3.1 EMA (Exponential Moving Average)

EMA는 최근 가격에 더 높은 가중치를 부여하는 이동평균이다.

**수식:**
```
Multiplier = 2 / (Period + 1)
EMA_today = (Price_today - EMA_yesterday) * Multiplier + EMA_yesterday
```

**Fast EMA(3):**
```
Multiplier = 2 / (3 + 1) = 0.5
EMA3_t = (Close_t - EMA3_{t-1}) * 0.5 + EMA3_{t-1}
```
- 가중치 50%가 최신 가격에 반영
- 가격 변동에 매우 빠르게 반응
- 단기 모멘텀 포착에 적합

**Slow EMA(100):**
```
Multiplier = 2 / (100 + 1) = 0.0198
EMA100_t = (Close_t - EMA100_{t-1}) * 0.0198 + EMA100_{t-1}
```
- 가중치 약 2%만 최신 가격에 반영
- 장기 트렌드의 방향을 안정적으로 표시
- 노이즈 필터링 효과 우수

**EMA(3)/EMA(100) 조합의 의미:**
- 극단적 비대칭 조합 (3 vs 100)
- Fast EMA가 매우 빠르게 반응하여 트렌드 초기 진입 가능
- Slow EMA가 장기 방향을 확인하여 역추세 거래 방지
- 크로스오버 빈도가 낮아 과매매(overtrading) 방지

### 3.2 ADX (Average Directional Index) - Wilder's Smoothing

ADX는 추세의 강도를 0~100 사이의 값으로 나타낸다. 방향은 알려주지 않고 **강도만** 측정한다.

**계산 과정:**

**Step 1: True Range (TR)**
```
TR = max(High - Low, |High - Close_prev|, |Low - Close_prev|)
```

**Step 2: Directional Movement (DM)**
```
+DM = High - High_prev    (양수이고 > -DM 일 때만, 아니면 0)
-DM = Low_prev - Low       (양수이고 > +DM 일 때만, 아니면 0)
```

**Step 3: Wilder's Smoothing (14-period 기본, 본 전략은 20-period)**
```
중요: Wilder's smoothing은 일반 EMA와 다르다.

ATR_20 = ATR_prev - (ATR_prev / 20) + TR_current
+DM_20 = +DM_prev - (+DM_prev / 20) + +DM_current
-DM_20 = -DM_prev - (-DM_prev / 20) + -DM_current
```

**Step 4: Directional Indicators**
```
+DI_20 = (+DM_20 / ATR_20) * 100
-DI_20 = (-DM_20 / ATR_20) * 100
```

**Step 5: DX (Directional Movement Index)**
```
DX = (|+DI_20 - -DI_20| / (+DI_20 + -DI_20)) * 100
```

**Step 6: ADX (Wilder's Smoothed DX)**
```
ADX_20 = ((ADX_prev * 19) + DX_current) / 20
```

**ADX >= 30 필터의 의미:**
```
ADX < 20  : 추세 없음 (횡보) -> 진입 금지
ADX 20~30 : 약한 추세 -> 진입 금지
ADX >= 30 : 강한 추세 -> 진입 허용
ADX >= 50 : 매우 강한 추세 -> 진입 허용
```

**Wilder's Smoothing 보정이 중요한 이유:**
- 일반 SMA/EMA와 다른 감쇠 방식 사용
- Period N의 Wilder's Smoothing = 2*N-1 기간의 EMA와 유사
- ADX(20)은 실질적으로 EMA(39)에 해당하는 smooth 효과
- 라이브러리마다 구현이 다를 수 있으므로 반드시 Wilder's 방식 확인 필요

### 3.3 RSI (Relative Strength Index)

**계산 과정:**

**Step 1: Price Change**
```
Change = Close_t - Close_{t-1}
Gain = max(Change, 0)
Loss = max(-Change, 0)
```

**Step 2: Average Gain/Loss (Wilder's Smoothing, 14-period)**
```
첫 번째: AvgGain_14 = sum(Gain, 14) / 14
이후:    AvgGain_14 = (AvgGain_prev * 13 + Gain_current) / 14

첫 번째: AvgLoss_14 = sum(Loss, 14) / 14
이후:    AvgLoss_14 = (AvgLoss_prev * 13 + Loss_current) / 14
```

**Step 3: RS & RSI**
```
RS  = AvgGain_14 / AvgLoss_14
RSI = 100 - (100 / (1 + RS))
```

**RSI 30~70 필터의 역할:**
```
RSI < 30  : 과매도 -> 숏 진입 금지 (반등 가능성)
RSI 30~70 : 중립 구간 -> 진입 허용
RSI > 70  : 과매수 -> 롱 진입 금지 (조정 가능성)
```

- 추세추종 전략에서 극단적 RSI 구간 진입을 방지
- 이미 과열된 시장에서의 추격매매 방지
- ADX와 결합 시 "강한 추세이지만 아직 과열되지 않은" 구간만 포착

## 4. 진입/청산 로직 상세

### 4.1 롱(Long) 진입 조건

```python
# 모든 조건이 동시에 충족되어야 진입
def check_long_entry(candles):
    ema3  = EMA(close, period=3)
    ema100 = EMA(close, period=100)
    adx20 = ADX(high, low, close, period=20)  # Wilder's method
    rsi14 = RSI(close, period=14)

    condition_1 = ema3[-1] > ema100[-1]      # EMA(3)이 EMA(100) 위로 교차
    condition_2 = ema3[-2] <= ema100[-2]      # 이전 봉에서는 아래 또는 동일
    condition_3 = adx20[-1] >= 30             # ADX >= 30 (강한 추세)
    condition_4 = 30 <= rsi14[-1] <= 70       # RSI 중립 구간

    return condition_1 and condition_2 and condition_3 and condition_4
```

**진입 시나리오:**
1. BTC 가격이 장기 하락 후 반등 시작
2. EMA(3)이 빠르게 상승하며 EMA(100)을 상향 돌파
3. ADX가 30 이상으로 추세 강도 확인
4. RSI가 30~70 사이로 아직 과열되지 않음
5. -> **롱 포지션 진입** (계좌의 70% 마진, 10x 레버리지)

### 4.3 청산 로직

#### 4.3.1 손절 (Stop Loss)
```
조건: ROI <= -8%
동작: 즉시 시장가 전량 청산
결과: 잔고의 약 5.6% 손실 (Margin 70% x 8% = 5.6%)
      레버리지 10x 적용 시 실질 손실 = 잔고의 56%
```

**주의: 실질 손실 계산**
```
포지션 크기 = 잔고 * 70% * 10x = 잔고 * 7배
SL -8% 발동 시 손실액 = 포지션 크기 * 8% = 잔고 * 56%
따라서 한 번의 SL이 잔고의 56%를 소멸시킨다.
```

#### 4.3.2 트레일링 스톱 (Trailing Stop)
```
활성화 조건: ROI >= +6%
트레일링 거리: 최고 ROI 대비 -5%

예시:
  ROI +6% 도달 -> 트레일링 활성화
  ROI +10% 도달 -> SL = +5% (10% - 5%)
  ROI +20% 도달 -> SL = +15% (20% - 5%)
  ROI +15%로 하락 -> SL = +15% (최고점 +20% 기준 유지)
  ROI +15% 도달 -> 청산 (최고 +20%에서 5% 하락)
```

#### 4.3.3 역신호 청산 (Reverse Signal Exit)
```
조건: 현재 포지션 방향과 반대되는 진입 신호 발생
동작:
  1. 현재 포지션 즉시 청산
  2. 반대 방향으로 신규 포지션 진입
```

## 5. 리스크 관리

### 5.1 포지션 사이징

```
계좌 잔고: B
마진 할당: B * 70% = 0.7B
레버리지: 10x
실질 포지션 크기: 0.7B * 10 = 7B (잔고의 7배)

예시) 잔고 $100,000
  마진: $70,000
  포지션 크기: $700,000
  1% 가격 변동 = $7,000 (잔고의 7%)
```

### 5.2 Position Exposure [(0.10, 0.25)]

```
PE(min) = 0.10: 최소 노출 비율 10%
PE(max) = 0.25: 최대 노출 비율 25%

이는 변동성에 따라 실제 투입되는 마진 비율이
전체 마진의 10%~25% 사이에서 동적으로 조절됨을 의미.

실질 레버리지 범위:
  최소: 70% * 10x * 10% = 0.7x (보수적)
  최대: 70% * 10x * 25% = 1.75x (적극적)
```

## 10. MDD 및 리스크 심층 분석

### 10.1 MDD 76.6%의 의미

```
Maximum Drawdown 76.6%는 전략 운용 기간 중
최고점 대비 최대 76.6%까지 잔고가 감소했음을 의미한다.

예시: 잔고 $100,000 -> $23,400 까지 감소
이후 회복하여 다시 $100,000 이상으로 상승

이는 매우 높은 MDD로, 실전 운용 시
심리적/재정적 견딜 수 있는 수준인지 반드시 검토해야 한다.
```

### 10.2 왜 SL 3회가 이렇게 큰 손실을 만드는가

**핵심 메커니즘: Margin 70% x Leverage 10x = 7x Exposure**

```
┌──────────────────────────────────────────────────────────┐
│  SL 발동 시 손실 계산                                      │
│                                                          │
│  잔고: B                                                  │
│  마진: B * 70% = 0.7B                                    │
│  포지션: 0.7B * 10x = 7B                                 │
│  SL -8% 손실: 7B * 8% = 0.56B                           │
│                                                          │
│  결론: 한 번의 SL이 잔고의 56%를 소멸시킨다                │
│                                                          │
│  2연속 SL 발생 시:                                        │
│  1차 SL: B -> 0.44B (56% 손실)                           │
│  2차 SL: 0.44B -> 0.1936B (추가 56% 손실)                │
│  누적 손실: 80.6%                                        │
│                                                          │
│  이것이 MDD 76.6%의 원인이다.                              │
└──────────────────────────────────────────────────────────┘
```

   - 향후 시장 구조가 변할 수 있다.

2. MDD 76.6%는 실전에서 더 클 수 있다.
   - 급격한 시장 변동(flash crash) 시 SL이 의도한 가격에 체결되지 않을 수 있다.
   - 슬리피지로 인해 실제 손실이 -8%를 초과할 수 있다.

   - 강제 청산(liquidation) 위험이 존재한다.
   - Isolated margin이므로 할당 마진 이상의 손실은 없지만,
     잔고의 56%가 한 번에 소멸할 수 있다.

4. 초기 자본이 클수록 슬리피지 영향이 커진다.
   - $3,000 기준 백테스트는 시장 영향(market impact)이 미미하다.
   - $100K+ 자본에서는 대량 주문 시 슬리피지가 증가한다.
   - BTC/USDT는 유동성이 높아 상대적으로 양호하지만,
     급변 시에는 예외이다.

5. 세금 및 규제 리스크
   - 국가별 암호화폐 세금 정책이 다르다.
   - 거래소 규제 변화 리스크가 있다.
   - 수익의 일부는 세금으로 납부해야 한다.

6. 운영 리스크
   - 서버 24/7 안정적 운영 필요
   - API 키 보안 관리 필수
   - 거래소 API 변경 시 코드 수정 필요
   - 네트워크 장애 대비 필요

### 17.2 WMA (Weighted Moving Average) 수식

```
WMA(n) = (P_1 * n + P_2 * (n-1) + ... + P_n * 1) / (n + (n-1) + ... + 1)

WMA(3):
  Weight sum = 3 + 2 + 1 = 6
  WMA = (Close_t * 3 + Close_{t-1} * 2 + Close_{t-2} * 1) / 6
```

### 17.4 ADX Wilder's vs Standard 차이

```
Standard EMA Smoothing:
  α = 2 / (N + 1)
  EMA_t = α * X_t + (1 - α) * EMA_{t-1}

Wilder's Smoothing:
  α = 1 / N
  WS_t = (1/N) * X_t + ((N-1)/N) * WS_{t-1}

차이점:
  - Wilder's는 EMA보다 더 느리게 반응
  - ADX(20) Wilder's ≈ EMA(39) Standard
  - 동일한 Period 입력 시 결과가 다름

주의: ta 라이브러리의 ADX는 Wilder's를 사용
      pandas_ta도 Wilder's 기본값
      직접 구현 시 반드시 Wilder's 방식 확인 필요
```
